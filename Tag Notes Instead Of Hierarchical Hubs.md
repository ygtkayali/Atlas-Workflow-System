# Tag Notes Instead Of Hierarchical Hubs

Status: [[status-pending]]
Parent: [[Idea Hub]]
Related: [[Note Manager]]
Created: 15-04-2026

## Summary

Possible future vault pattern: use empty notes as classification tags instead of relying on `Main Hub` style notes that can be interpreted as hierarchy.

In this model, a tag is an empty note referenced through normal `[[...]]` links rather than `#` tags. The purpose is discovery and classification, not content ownership or hierarchy.

The main motivation is not richer discovery by itself. The main motivation is reducing the tendency for agents to interpret `Main Hub` and `Parent` as a hierarchical note-taking structure.

## Details

This idea treats category-like notes as lightweight classification anchors.

Example direction:
- `Idea Hub` could become something closer to an `idea_hub` style classification note
- notes could link to that tag-like note for search and grouping
- the note should be understood as a criterion or category, not as a parent in a strict hierarchy

This may be more useful in a broad knowledge vault where multiple domains intersect and many notes benefit from shared discovery points.

For this project, it is still unclear whether that additional structure is actually needed. Technical project notes may already form enough useful links through normal note creation and implementation workflows.

This idea should remain speculative for now. It is not an approved redesign of the vault.

## Open Questions

- Is the real issue vault structure, or just agent interpretation of existing fields like `Main Hub` and `Parent`?
- Would clearer instructions and template semantics remove most of the problem without introducing tag notes?
- Should classification notes supplement the current system or replace some existing note roles later?
- Does a technical-project vault benefit enough from this pattern to justify the added complexity?

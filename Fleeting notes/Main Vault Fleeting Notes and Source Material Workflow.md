## Idea

The main vault has `Fleeting notes` and `Source material` folders that should remain meaningful for human readability and knowledge work.

`Fleeting notes` should stay distinct, but the notes inside that folder should be consistent with the Idea Note structure used in this workflow.

`Source material` should hold book and article notes. Those notes may later promote new full notes or update existing full notes after the material has been processed.

## Questions

- Should fleeting notes use the same metadata and question-oriented shape as Idea Notes?
- Should source material notes have their own template separate from Idea Notes and General Notes?
- What marks a source material note as ready to promote into a full note or update an existing full note?
- Should promotion from source material to full notes route through `clarify-intent` and `Note Manager`?
- How should agents preserve the difference between raw extraction, interpretation, and settled durable knowledge?

## Current Thinking

- Fleeting notes can use the Idea Note structure while remaining in a human-readable fleeting folder.
- Source material notes should not be treated as settled knowledge by default.
- Promotion should probably be explicit: source material produces candidate insights, then `clarify-intent` shapes them, and `Note Manager` decides durable note creation or updates.

## Open Decisions

- Whether source material notes need a dedicated status or metadata field.
- Whether promoted insights should backlink to the source material note.
- Whether the main vault needs a specific source-processing playbook.

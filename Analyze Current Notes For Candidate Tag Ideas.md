## Summary

Analyze the current notes in this repository and propose a useful candidate tag set.

The main output should be a ranked top 10 list of tags that would improve discovery and classification without silently redesigning the vault.

## Details

This task should stay bounded to the current repository notes and current workflow direction.

The goal is not to replace the existing note-role model or automatically retag the vault.
The goal is to inspect what kinds of subjects and retrieval needs are already showing up, then suggest a small tag set that would actually help.

The analysis should treat the current status tags as a separate layer unless there is a strong reason to compare them explicitly with new candidate tags.

One concrete example already surfaced while reviewing the current notes:
- there may be a useful classification need around better note types or note-type quality

That example should be treated as a seed, not as a decided solution.
It may point toward tags around note roles, note structure, or note-model concerns rather than a literal final tag.

Useful output shape:
- ranked top 10 candidate tags
- short rationale for each tag
- example notes each tag would apply to
- brief recommendation on whether the tags should be normal tags, note-based classification notes, or remain undecided for now

## Open Questions

- Should the top 10 exclude the existing status tags entirely, or compare them with new candidates?
- Should the result recommend normal tags, note-based classification notes, or both?
- Is the real need better retrieval, better note types, or clearer note-structure semantics?
- Should the output also recommend any follow-up note changes if the tag ideas are strong enough?
